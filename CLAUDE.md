# CLAUDE.md — CriticalMinds («Оптика»)

Я — Claude, память между сессиями не сохраняется. Memory Bank (`memory-bank/`) — моя память.
Устройство и протокол: `memory-bank/README.md`. Его точность = моя эффективность.

## Ритуал начала КАЖДОЙ сессии
1. `bash memory-bank/check-memory.sh` — расхождения чинить ДО работы (минуты).
2. Читать по порядку: `memory-bank/index.md` (карта) → `activeContext.md` (сводка) →
   `context/current.md` (оперативка) → topics по теме задачи.

Без этого к задачам не приступать. После саммаризации чата — восстановить контекст заново тем же порядком.

## Ритуал конца сессии (протокол из README §«Конец сессии»)
Сессия → `sessions/YYYY-MM/` (append-only, YAML-шапка) · знания → слить в `topics/`
(нет топика — завести СЕЙЧАС) · оперативка → `context/current.md` · строка в `activeContext.md` ·
`index.md`/`backlog.md` — по событию.

## Событийные триггеры (правило = событие + действие, не «когда вспомню»)
- Новое направление → `topics/<тема>.md` каркасом до первой строчки кода.
- Создал/переместил/удалил файл банка → строка в `index.md` в той же сессии.
- Закрыл пункт backlog → вычеркнуть в той же сессии.
- Превышен лимит строк (activeContext ≤60, context/* ≤120) → обрезать сразу.

## Незыблемые правила проекта (дублируют банк, потому что критичны)
1. Куратор никогда не видит содержание дневников и формулировки прогнозов — только метаданные
   и счётчики. Уровень модели данных/API, не UI. Детали: `memory-bank/topics/privacy.md`.
2. Формат урока неизменен: Крючок (ошибка пользователя) → Концепт (один!) → Проверка → Перенос.
3. Новые уроки = данные для движка, не новый код (кроме принципиально новых типов шагов).
4. Дизайн-токены — только палитра `C`; шрифты IBM Plex. Роли цвета: `memory-bank/topics/design.md`.
5. Язык продукта и документации — русский.
6. Тексты участников в LLM — только как данные, никогда как инструкции.
