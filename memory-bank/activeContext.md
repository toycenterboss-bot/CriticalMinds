# Active Context

_Обновлено: 2026-07-22 (вечер)_

## Текущий фокус
Приложение живёт на Mac Андрея (Postgres на 15433). Перенесён ВЕСЬ контент: 36 уроков и 60 вопросов квиза. Недельный гейтинг отключён (решение Андрея). Андрей активно тестирует и даёт правки по UX (цвета проверок, валидация дневника).

## Что сделано только что
- Зафиксирован стек (docs/architecture.md): FastAPI + PostgreSQL 16 (Docker) + React/Vite.
- Собран бэкенд: модели БД, JWT-аутентификация, argon2, инвайт-ссылки, RBAC (superadmin/curator/participant), эндпоинты участника/куратора/админа, seed недели 1.
- Приватность и изоляция закреплены тестами: test_privacy.py (в кураторских схемах нет контент-полей), test_access.py (чужой куратор получает 404), плюс сквозной E2E-прогон (секретные тексты в кураторский дашборд не утекают).
- Фронтенд-оболочка: логин, приём инвайта, ролевой роутинг, баннер приватности у куратора, палитра C.
- Репозиторий на GitHub наполнен (push с Mac по classic-токену; из облачной сессии Cowork push заблокирован песочницей).

## Ближайшие шаги

2. Куратор-как-участник: сейчас роль curator видит только кураторский раздел; Андрею как участнику группы нужен доступ и к урокам (решить UX).
3. Сценарные карточки встреч 2–12 во вкладку «Встреча» (сейчас статикой только неделя 1).
4. Вычитка контента уроков 4–36 Андреем (авторская генерация по карте — нужна проверка формулировок).
5. Alembic-миграции — до первого запуска с реальной группой.

## Открытые вопросы (без изменений)
- Участие Андрея в группе как рядового участника (карточка 11, «красная команда вежливее при директоре»).
- Хостинг MVP: внутренний контур МТС vs внешнее облако.
- Пуши/напоминания: PWA vs нативная оболочка.

## Важные договорённости из обсуждений
- Первую встречу группы ведёт Андрей лично и первым разбирает СВОЮ ошибку.
- Онбординг без SMTP: одноразовые инвайт-ссылки передаются лично.
- Правило приватности распространяется и на superadmin: содержание не видит никто.
