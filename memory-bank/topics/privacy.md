# Topic: Приватность
> Заведён 2026-07-23. Самая охраняемая инварианта проекта — условие честности записей участников.

## Состояние
Реализована на уровне API и закреплена автотестами. Нарушений не зафиксировано.

## Железные правила
1. Тексты дневников и формулировки прогнозов не видит НИКТО, кроме автора: ни куратор, ни superadmin. На встречу попадает только помеченное «поделиться».
2. Барометр настроения: индивидуальные значения не покидают участника; куратору — агрегат группы при ≥3 ответивших (k-анонимность, `curator.py::_mood_aggregate`, MIN_MOOD_RESPONSES=3).
3. Журнал встреч — осознанно ГРУППОВОЙ артефакт (совместные выводы), виден группе и куратору; личные материалы туда не попадают.

## Как обеспечено (механизмы, не обещания)
- Отдельный блок кураторских Pydantic-схем (`schemas.py`: MemberMeta, GroupDashboard…) без единого контент-поля — контент физически не сериализуется.
- Кураторские SQL-запросы используют только COUNT/MAX-агрегаты; текстовые колонки не попадают даже в SELECT.
- `tests/test_privacy.py` валит сборку, если в кураторской схеме появляется поле из чёрного списка (text, content, body…); `tests/test_access.py` — изоляция групп.
- flags.py не обращается к текстовым полям (проверяется тестом по исходнику).

## Правило на будущее (LLM)
Тексты участников в LLM сейчас не передаются. Если появятся LLM-фичи — тексты подставлять только как ДАННЫЕ, никогда как инструкции (защита от prompt-инъекций через дневник).
