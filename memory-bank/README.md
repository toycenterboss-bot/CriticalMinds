# Memory Bank «Оптики» — протокол ведения

> Адаптация переносимого чертежа (оригинал: `archive/memorybankblueprint.md`).
> Главная идея: у ассистента нет памяти между сессиями — memory bank и есть его память.
> Качество памяти определяется дисциплиной: каждому виду знания — своё место,
> каждому правилу — событие-триггер.

## Четыре вида информации — четыре места

| Тип | Живёт | Пример | Срок жизни |
|---|---|---|---|
| Оперативка | `context/current.md` (≤120 строк) | «правлю X, остановился на Y, дальше Z» | дни |
| Знания | `topics/` (перезаписываются, не растут в хвост) | «как устроен гейтинг, какие грабли» | месяцы |
| План | `plans/` (закрытые → `plans/closed/` после харвеста) | «что и в каком порядке в задаче X» | недели |
| История | `sessions/YYYY-MM/` (append-only) | отчёт сессии с YAML-шапкой | вечно |

Навигаторы: `index.md` (карта, читается ПЕРВЫМ), `activeContext.md` (сводка ≤60 строк),
`backlog.md` (очередь задач). Слоты не используются: сессия одна, оперативка в `context/current.md`.

**Тест на тип:** фраза «как устроено / почему так / правило / ⚠️», переживающая текущую
задачу, — это знание → `topics/`, не оперативка.

## Начало сессии

1. `bash memory-bank/check-memory.sh` — расхождения чинить ДО работы.
2. Читать: `index.md` → `activeContext.md` → `context/current.md` → topics по задаче.

## Конец сессии (протокол)

1. **Сессия → архив:** `sessions/YYYY-MM/YYYY-MM-DD-slug.md` с YAML-шапкой
   (`date`, `topics`, `commits`, `итог`) и отчётом.
2. **Знания → topics:** слить в существующий текст (НЕ в хвост).
   Нет подходящего топика — завести СЕЙЧАС.
3. **Оперативка → `context/current.md`:** где остановился, следующий шаг.
4. **Сводка одной строкой** в `activeContext.md`.
5. `index.md`/`backlog.md` — по событию (см. триггеры).

## Событийные триггеры (правило = событие + действие)

- Открыл новое направление → завести `topics/<тема>.md` каркасом СРАЗУ.
- Создал/переместил/удалил файл банка → строка в `index.md` в ТОЙ ЖЕ сессии.
- Направление закрыто → выжимка в `topics/`, план в `plans/closed/`,
  `context/current.md` очистить до «что дальше».
- Превышен лимит строк → обрезать сразу, не «когда-нибудь».
- Закрыл пункт backlog → вычеркнуть в ТОЙ ЖЕ сессии.

## Разделение труда

Машина (`check-memory.sh`) проверяет то, что видно из файлов: полнота index, лимиты строк,
битые ссылки, свежесть backlog. Человек — то, что из файлов не видно (закрыта ли задача).
Текстовое правило остаётся только там, где машина бессильна.
