# System Patterns

## Архитектура прототипа (текущая)
Один React-компонент (`prototype/optika_prototype.jsx`), состояние в памяти (useState), без бэкенда. Цель прототипа — проверка механики уроков и UX, не продакшен.

## Ключевые паттерны

### 1. Урок как данные + движок
Уроки описываются декларативно в массиве `LESSONS`: урок = массив шагов с типами. Движок `LessonPlayer` рендерит шаг по `step.type`:
- `hookNumber` — крючок с числовым вводом (ловушка/верный ответ, разные reveal-тексты)
- `hookChoice` — крючок с выбором варианта и общим reveal
- `concept` — теория (title + body, абзацы через \n\n)
- `check` — проверка с мультивыбором, фидбек per-option; опционально `labels` для бинарной разметки (факт/интерпретация)
- `firstEntry` — встроенная форма первой записи дневника (урок 3)
- `transfer` — задание-перенос с меткой для дневника

**Добавление новых уроков (недели 2–12) = добавление данных, не кода.** Новые типы шагов добавлять в движок только если формат принципиально новый.

### 2. Калибровка как сквозная сущность
Все источники уверенности стекаются в один график (`CalibrationChart`):
- квизы → точки при заявленных 90%
- разрешённые прогнозы → бакеты по уровням уверенности
Диагональ = идеал. Паттерн: любая новая механика с процентами уверенности обязана попадать на этот график.

### 3. Приватность на уровне модели данных
Кураторский раздел получает ТОЛЬКО метаданные: `{lessons: [id, день, минуты], journal: count, shared: count, preds: count, quizHits}`. Тексты записей в кураторские структуры не передаются вообще (не «скрыты в UI», а отсутствуют в данных). При переходе на бэкенд это должно стать правилом API: эндпоинты куратора физически не возвращают content-поля.

### 4. Автофлаги внимания
`teamFlags()` — чистая функция над метаданными. Уровни: risk / warn / info. Текущие правила: урок ≤5 мин (пролистан), уроки без записей в дневнике, неактивность 2+ дня, квиз не пройден, ноль прогнозов. Новые правила добавлять сюда же, не размазывать по UI.

### 5. Дизайн-система
Токены в объекте `C` (палитра «лабораторный журнал»: бумага в клетку, чернила, теал, маркер). Шрифты IBM Plex Sans / IBM Plex Mono (кириллица). Моноширинный — для данных/чисел/меток, гротеск — для текста. Компоненты: `Btn`, `Tag`, `Card`. Фон — миллиметровая сетка (CSS gradients). Не вводить новые цвета вне палитры.
