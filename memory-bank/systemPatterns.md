# System Patterns

## Архитектура прототипа (текущая)
Один React-компонент (`prototype/optika_prototype.jsx`), состояние в памяти (useState), без бэкенда. Цель прототипа — проверка механики уроков и UX, не продакшен.

## Ключевые паттерны

### 1. Урок как данные + движок
Уроки описываются декларативно в массиве `LESSONS`: урок = массив шагов с типами. Движок `LessonPlayer` рендерит шаг по `step.type`:
- `hookNumber` — крючок с числовым вводом (ловушка/верный ответ, разные reveal-тексты)
- `hookChoice` — крючок с выбором варианта и общим reveal
- `concept` — теория (title + body, абзацы через \n\n)
- `check` — проверка с мультивыбором, фидбек per-option; опционально `labels` для бинарной разметки (факт/интерпретация)
- `firstEntry` — встроенная форма первой записи дневника (урок 3)
- `transfer` — задание-перенос с меткой для дневника

**Добавление новых уроков (недели 2–12) = добавление данных, не кода.** Новые типы шагов добавлять в движок только если формат принципиально новый.

### 2. Калибровка как сквозная сущность
Все источники уверенности стекаются в один график (`CalibrationChart`):
- квизы → точки при заявленных 90%
- разрешённые прогнозы → бакеты по уровням уверенности
Диагональ = идеал. Паттерн: любая новая механика с процентами уверенности обязана попадать на этот график.

### 3. Приватность на уровне модели данных
Кураторский раздел получает ТОЛЬКО метаданные: `{lessons: [id, день, минуты], journal: count, shared: count, preds: count, quizHits}`. Тексты записей в кураторские структуры не передаются вообще (не «скрыты в UI», а отсутствуют в данных). При переходе на бэкенд это должно стать правилом API: эндпоинты куратора физически не возвращают content-поля.

### 4. Автофлаги внимания
`teamFlags()` — чистая функция над метаданными. Уровни: risk / warn / info. Текущие правила: урок ≤5 мин (пролистан), уроки без записей в дневнике, неактивность 2+ дня, квиз не пройден, ноль прогнозов. Новые правила добавлять сюда же, не размазывать по UI.

### 5. Дизайн-система
Токены в объекте `C` (палитра «лабораторный журнал»: бумага в клетку, чернила, теал, маркер). Шрифты IBM Plex Sans / IBM Plex Mono (кириллица). Моноширинный — для данных/чисел/меток, гротеск — для текста. Компоненты: `Btn`, `Tag`, `Card`. Фон — миллиметровая сетка (CSS gradients). Не вводить новые цвета вне палитры.

## Решения и их причины (log)
- **2026-07-22:** Стрик считается неделями, не днями — взрослая аудитория, жёсткие стрики вызывают отвал.
- **2026-07-22:** Кураторские флаги сопровождаются рекомендацией «личный разговор, не общий чат» прямо в UI — снижение риска токсичного применения метрик.
- **2026-07-22:** Первая запись дневника встроена в урок 3 как шаг — конверсия в первую запись важнее чистоты навигации.
- **2026-07-22 (арх. MVP):** Стек зафиксирован: FastAPI + PostgreSQL 16 (Docker) + React/Vite — знакомый минимум без BaaS; СУБД настоящая с первого дня по требованию Андрея.
- **2026-07-22 (арх. MVP):** Приватность контента реализована как отдельный набор кураторских Pydantic-схем без текстовых полей + автотест test_privacy.py; кураторские SQL-запросы используют только COUNT/MAX-агрегаты.
- **2026-07-22 (арх. MVP):** Изоляция групп: чужой куратор получает 404 (не 403), чтобы не раскрывать факт существования группы.
- **2026-07-22 (арх. MVP):** Онбординг — одноразовые инвайт-ссылки (hash в БД, TTL 72 ч) вместо SMTP: паролей в чатах нет, почтовый сервер не нужен.
- **2026-07-22 (вечер):** Недельный гейтинг уроков отключён по решению Андрея — все уроки доступны сразу (available=True в /me/lessons). Механика current_week сохранена для квизов и дашборда.
- **2026-07-22 (вечер):** Добавлен 7-й тип шага hookText (свободный ввод + reveal) — понадобился урокам с рефлексией. Контент остальных 35 уроков не потребовал нового кода — паттерн «урок = данные» подтверждён.
- **2026-07-22 (вечер):** Валидация пользовательских текстов централизована в schemas.clean_text: длины, минимум букв, разнообразие символов, вычистка невидимых/управляющих. Правило на будущее: тексты участников в LLM подставлять только как данные, не как инструкции.
- **2026-07-23:** Гейтинг v2 по решению Андрея: неделя N+1 открывается после завершения недели N (все уроки + квиз). Календарный гейтинг убран совсем.
- **2026-07-23:** Мотивационные механики (расширение брифа по явному решению Андрея): вехи, пульс группы, «первопроходец», турнир калибровки. Принцип: себя участник видит явно, других — только анонимно/агрегированно; адресного «вы отстали» нет.
- **2026-07-23:** Барометр настроения: индивидуальные значения не покидают участника; куратору — только агрегат при ≥3 ответивших (_mood_aggregate). Расширение принципа «количество, не содержание» на самоотчёты о состоянии.
- **2026-07-23:** Оффлайн-задания недель = данные (content/materials.py → week_materials). Ссылки только на стабильные источники (Википедия); отметка выполнения не влияет на гейтинг, куратору — только счётчик.
